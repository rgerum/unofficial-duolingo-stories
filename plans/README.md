## Findings considered and rejected

(So nobody re-audits these.)

From the 2026-06-11 audit:

- **"Committed production secrets in `.env.local`"** — false: the file is gitignored (`.gitignore:25,56`) and has never been committed (`git log --all -- .env.local` is empty). Note: real secrets do live in the local file; an audit subagent echoed some values into the 2026-06-11 advisor session transcript (stored under `~/.claude/`), so precautionary rotation was suggested to the maintainer.
- **"Permanent completion-flag lock breaks story completion"** (`story_wrapper.tsx:111-126`) — by design: an idempotency guard; the ref resets when the next story mounts. Known micro-edge: if navigation fails after a successful completion, the finish button becomes a no-op for that view. Not worth a plan.
- **"Check-then-act race in `storyWrite.ts:140-151`"** — not a race: Convex mutations are serializable transactions with optimistic concurrency control.
- **"Audio-cutter dialog bloats other routes' bundles"** — wrong: it is imported only by its own route (`audio-cutter/page_client.tsx:9`), which Next.js code-splits. Reframed as the maintainability extraction in plan 009.
- **"Attacker-corruptible Discord role-sync snapshots"** (`discordBot.ts:228+`) — requires the same shared secret as everything else on that trust boundary; subsumed by plan 008.
- **lucide-react / lamejs version lag** — cosmetic / stable-as-is; not worth the regression risk in audio encoding. Revisit only when a concrete need appears.
- **Validator duplication across `adminData`/`landing`/`editorRead`** — real but low-value consolidation; shapes change rarely. Reconsider if a shared shape changes twice in one quarter.
- **Minor perf items** — sequential `ctx.db.get` loops (`storyDone.ts` — usually exits on first row; `editorRead.ts` — legacy fallback path), `landing.ts` per-language localization collects (tables are small), `getEditorSidebarData` full-course collect (small table; also gated by plan 002 after it lands). Not worth plans at current data sizes.
- **Unhandled rejection in `DocsSearchModal.tsx:109`** — plausible but unvetted in depth and low-blast-radius (docs search init). Fix opportunistically when touching that file.
- **`as any` casts in `convex/authMigration.ts` / `authFunctions.ts`** — migration-era code scheduled to age out with plan 010's outcome; not worth independent cleanup.

Added by the 2026-07-09 re-audit:

- **`splitStoryElementsIntoParts` sentinel `-1` collides with a representable `line_index`** (`src/lib/story-parts.ts:7-13` vs `syntax_parser_new.ts:868`) — latent only: reachable if a `MULTIPLE_CHOICE` at `line_index -1` ever leads the element list, which current parser output can't produce (stories start with a title at index 0). One-line guard; fix opportunistically when touching the file.
- **`serve-local-story-audio.ts` path traversal** — checked and adequately contained: normalizes, strips `..`, and verifies `filePath.startsWith(root)`; local dev tool.
- **Feedback content stored-XSS** — checked and absent: rendered only through React text nodes (`feedback_review_view.tsx:163-179`); no `dangerouslySetInnerHTML`.
- **Expo SDK one major behind (56 vs 57)** — track, don't act: schedule a dedicated upgrade branch with device smoke tests when SDK 56 approaches end of support; do not bundle into feature work.
- **Mobile `story_done` writes trusting client identity** — checked and absent: identity is derived server-side from the Convex session in every mutation the mobile app calls.

## Decided by the maintainer (do not re-raise)

- **Account deletion keeps editorial data** (2026-07-09): `story_approval` rows and `courses.contributorDetails` attribution survive account deletion. Rationale: deleting approvals would break the editor flow — it becomes unclear why a story is published if the approvals that finished it vanish later.
