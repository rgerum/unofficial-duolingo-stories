## Why this matters

`pnpm audit --prod` reports two HIGH advisories in transitive dependencies that sit on reachable production paths, and neither can be fixed by a routine `pnpm update` because the vulnerable versions are pinned by intermediate packages:

- **hono 4.12.23** (GHSA-88fw-hqm2-52qc — CORS middleware reflects any Origin with credentials; fixed in ≥4.12.25), reached via `@convex-dev/better-auth > convex-helpers > hono`. Reachable: `convex/http.ts` calls `authComponent.registerRoutes(http, createAuth)`, which mounts the Better Auth HTTP routes on this hono instance.
- **undici 6.26.0** (GHSA-vxpw-j846-p89q — WebSocket client DoS; fixed in ≥6.27.0), reached via `@vercel/blob > undici`. Reachable: `@vercel/blob` is imported in `src/app/audio/upload/route.ts` and the TTS provider modules under `src/app/audio/_lib/audio/`.

Separately, `app-mobile/package.json` exact-pins `@convex-dev/better-auth` to `0.12.2` (the web app uses `^0.12.4`), so auth fixes published in 0.12.x never reach the mobile app automatically. Auth is the one subsystem both apps genuinely share against the same Convex backend.

## Steps

### Step 1: Record the baseline

Run `pnpm audit --prod` at the repo root and save the output. Confirm it shows the two HIGH advisories for `hono` (path `.>@convex-dev/better-auth>convex-helpers>hono`) and `undici` (path `.>@vercel/blob>undici`).

**Verify**: both HIGH rows present. If they are already gone, STOP — the parents may have been bumped independently and this plan is stale.

### Step 2: Add the overrides

In `pnpm-workspace.yaml`, extend the existing `overrides:` block:

```yaml
overrides:
  kysely: 0.28.17
  hono: ">=4.12.25"
  undici: ">=6.27.0"
```

Then run `pnpm install`.

**Verify**: `pnpm install` exits 0; `pnpm why hono | head -20` shows a hono version ≥4.12.25; `pnpm why undici | head -20` shows undici ≥6.27.0.

### Step 3: Confirm the advisories are cleared

**Verify**: `pnpm audit --prod` → zero HIGH advisories (moderates may remain; that's expected).

### Step 4: Smoke-verify the affected surfaces build

Run `pnpm typecheck && pnpm lint && pnpm test`.

**Verify**: all exit 0. (These advisories live under auth HTTP routes and the audio upload path; the type/test gate is the cheapest regression signal available without a deployment. Note in your report that a manual login + audio-upload smoke test on a dev deployment is recommended before release.)

### Step 5: Unpin the mobile auth dependency

In `app-mobile/package.json`, change:

```json
"@convex-dev/better-auth": "0.12.2"
```

to a caret range matching the web app's floor:

```json
"@convex-dev/better-auth": "^0.12.4"
```

Then run `pnpm --dir app-mobile install`.

**Verify**: exit 0, and `grep '"@convex-dev/better-auth"' app-mobile/package.json` shows `^0.12.4`.

### Step 6: Mobile typecheck

**Verify**: `pnpm --dir app-mobile typecheck` → exit 0. If the 0.12.x bump introduces type errors in `app-mobile/src/auth-client.ts` or callers, STOP and report the errors verbatim — do not adapt source code in this plan.
